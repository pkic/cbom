# Purpose and status of this document
This document is an initial input to the PKI Consortium CBOM Profiles Working Group. It has two objectives. First, it sets out the justification for CBOM profiles as a distinct artifact and discipline. Second, it identifies the aspects the Working Group should consider defining to produce its principal output: a specification describing how to create a CBOM profile.
It is not the specification itself. It is a scoping document, intended to help the group agree on the shape of the work, divide it into tractable parts, and establish shared terminology before drafting begins. Each aspect in Section 3 is presented with a short rationale and the decisions the group will need to take.
The Working Group charter provides the mandate for this work: a vendor-neutral, format-independent methodology for defining CBOM profiles that can be expressed in either CycloneDX or SPDX, without becoming entangled in the governance of any single standards body. This document operates within that mandate and assumes it as given.
Intended outcome. A methodology that any sector or organisation — telecommunications, financial services, healthcare, government, or a single enterprise — can follow to define a CBOM profile fit for its purpose, with the Working Group’s methodology serving as the canonical reference for doing so.
